# Data Integration Platform - Claude Instructions

## Project Overview
Multi-Source Data Integration Platform with plugin architecture.

## Git Commit Rules
- After every new file created: commit with descriptive message
- After every feature completed: commit with summary
- Commit message format: [TYPE] Description
  - Types: FEAT, FIX, DOCS, TEST, REFACTOR
- Always push to main branch after committing

## Documentation Rules
- Every class must have a docstring explaining what it does
- Every function must have docstring with Args and Returns
- After every major component built, update README.md
- Keep a CHANGELOG.md updated with every change

## Code Standards
- Python 3.11
- Type hints on every function
- Poetry for dependency management
- pytest for all tests
